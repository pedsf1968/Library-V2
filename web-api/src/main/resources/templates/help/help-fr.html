Projet de système d'information de la bibliothèque d'une grande ville.

Ce RELEASE est constitué de l'application WEB pour les utilisateurs et d'un batch qui effectue des relances.

Tout utilisateur peut consulter les différents médias présents dans la bibliothèque.

Il y a quatre différents types de média: livre, jeux, musique et vidéo

Un utilisateur peut s'inscrire pour devenir membre et emprunter des médias

Un membre peut consulter ses emprunt

Un membre peut réserver un média pour pouvoir le retirer et l'emprunter

Un membre peut prolonger son emprunt quand il arrive a échéance

Un mail de relance est envoyé pour chaque retard.

MEDIA

C'est le type générique de ce qui peut être emprunter.

Il est caractérisé par :

  • Un code barre
  • Un titre
  • Une date de publication
  • Un stock
  • Une quantité restante

BOOK

C'est le type dérivé pour gérer les livres.

Il est caractérisé en plus par :

  • Un code ISBN
  • Un auteur
  • Un éditeur
  • Un type
  • Un format
  • Un nombre de pages
  • Un résumé

GAME

C'est le type dérivé pour gérer les jeux.

Il est caractérisé en plus par :

  • Un éditeur
  • Un type
  • Un format
  • Le code PEGI
  • Le lien vers une vidéo publicitaire
  • Un résumé

MUSIC

C'est le type dérivé pour gérer l'audio.

Il est caractérisé en plus par :

  • Un auteur
  • Un compositeur
  • Un interprète
  • Une durée en minutes
  • Un type
  • Un format
  • Le lien vers un clip vidéo d'une des chansons

VIDEO

C'est le type dérivé pour gérer les vidéo.

Il est caractérisé en plus par :

  • Un directeur
  • Une durée en minutes
  • Un type
  • Un format
  • La qualité de l'image
  • La qualité de du son
  • Le type d'audience
  • Le lien vers la bande annonce
  • Un résumé
  • La liste des acteurs
